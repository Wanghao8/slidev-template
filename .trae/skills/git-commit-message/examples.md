# 提交信息示例

## 示例 1：功能新增

**暂存变更**：新增登录页与 JWT 校验中间件

**上一提交**：`AI: #1234567 修复弹窗没有关闭按钮`

**生成结果**：
```
AI: #1234567 新增登录页与 JWT 校验
```

---

## 示例 2：Bug 修复

**暂存变更**：修正报表中时区导致的日期显示错误

**上一提交**：`AI: #1234567 修复弹窗没有关闭按钮`

**生成结果**：
```
AI: #1234567 修复报表日期时区显示错误
```

## 示例 3：无法提取 ID

**暂存变更**：更新 README 与依赖版本

**上一提交**：`feat: add docker config`（无 #ID 格式）

**生成结果**：
```
AI: #000000 更新 README 与依赖版本
```

## 示例 4：多类改动取重点

**暂存变更**：修复支付接口超时、顺带调整部分文案与样式

**生成结果**（优先突出核心修复）：
```
AI: #1234567 修复支付接口超时
```

正文可补充：同步调整相关文案与样式。

---

## 反例

| 错误 | 正确 |
|------|------|
| 修复了弹窗问题 | 修复弹窗关闭按钮 |
| 我添加了登录功能 | 新增登录功能 |
| AI: #1234568 …（ID 自增） | AI: #1234567 …（复用上条 ID） |
| 标题超过五十个字并且写得很长… | 拆成标题 + 正文 |
